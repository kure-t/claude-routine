# Claude Code 運用ガイド

## ルーティン作業の原則

タイムアウトを防ぐため、以下の順序を必ず守ること。

1. **ファイル書き込みを最初に実行する** — Markdown などの長文コンテンツはチャットに出力せず、直接ファイルに書き込む
2. **チャット出力は簡潔に** — 作業内容の説明は1〜2文に留め、コンテンツ全文をチャットに出力しない
3. **commit・push はファイル書き込み直後に実行する** — 確認ステップを挟まずそのまま続行する
4. **ユーザーへの確認は最小限に** — ルーティン作業中に「出力してから確認」フローを挟まない
